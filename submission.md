# SUBMISSION.md
My assignment for Week 3 has been to create a Research Desk agent from the Week 2 chatbot I developed earlier. To make this agent work as a research assistant, I built support for searching the web with Serper, searching through academic papers with the Hugging Face Papers API, reading the content of those papers, managing research notes with file tools, and tracking conversations across sessions. I also built a command-line REPL interface and a Textual-based user interface so that users can choose how they would like to interact with the agent.
The Agent class is where all of the agent functionality is implemented. Each agent maintains a conversation history which it passes to the language model along with definitions of any tools that are available for the current session. If the language model determines that a tool needs to be called, it will return a tool call as opposed to a response; therefore, the agent calls the function associated with that particular tool, appends the result to the conversation history, and then calls the language model again until the language model returns a normal response. This process allows the agent to execute multi-step tasks such as searching for papers, reading a paper, and saving the results to a note file before responding to the user.
Separating the project into many modules consisting of web tools, paper tools, file tools, and session management helped organize my project better than if they were all stored as one complete piece of code, making it easier to debug each of the modules. I was able to implement the Agent class in both the REPL Interface and the Textual UI without writing the code for both classes, enabling reuse of one class for two interfaces.
Also, making session persistence work correctly required a significantly larger amount of time than I had anticipated. My initial implementation of session persistence attempted to save the entire message history as a direct serialisation of the messages to JSON; however, due to some of the OpenAI SDK messages being non-serialisable, there were many errors when attempting to save the session. A workaround for this issue was to create a serializer for the messages that converted the message into a dictionary prior to being written to disk. Another issue related to persistence of messages confused me when some bot assistant messages contained None values; these needed special handling in order to save conversation history correctly.
TheImproving the session management system through multiple named sessions and the possibility of users switching sessions from the REPL are key features I would add or improve if I had any additional time. In addition, I would enhance the research workflow through better citation formats and improved paper/web search differentiation. Finally, I would work to polish the textual interface to make it easier for users to find previously saved research notes directly from the user interface.
Overall, this project has taught me that modern AI agents do much more than just chatbots. I learned how tool creating, session managements/external API's combined with a good user interface can create a more capable research assistant.
